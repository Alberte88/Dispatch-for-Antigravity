# Dispatch for Antigravity 🚀

> Control your local Antigravity AI coding agent from your phone via Telegram.

A self-hosted Node.js bot that bridges Telegram messages to the `agy.exe` CLI on your PC. Vibe-code from anywhere — your couch, a coffee shop, or mid-commute.

---

## Features

| Command | What it does |
|---|---|
| `<your PIN>` | Unlock the bot (20-min sliding session) |
| `/run <task>` | Start an Antigravity task (plain English!) |
| `/c <task>` | Continue previous Antigravity session |
| `<any text>` | Same as `/run` once unlocked |
| `/stop` | Kill the running task + all child processes |
| `/status` | Show bot state (locked/unlocked/running) |
| `/lock` | Instantly lock the session |
| `/workspace` | Show current working directory |
| `/workspace <path>` | Switch workspace (security-validated) |
| `/model` | Show current AI model |
| `/model <name>` | Switch model in settings.json |
| `/help` | Show all commands |

## Security

- **PIN gate** — all commands require a SHA-256 hashed PIN stored in `.env`
- **20-minute sliding timeout** — auto-locks after inactivity; resets on each task
- **Manual lock** — `/lock` revokes access instantly
- **Allowlist** — only your `ALLOWED_CHAT_ID` can interact with the bot
- **No command injection** — `shell: false` throughout; args passed as strict arrays
- **Path traversal protection** — workspace changes are validated against `WORKSPACE_ROOT`
- **Timing-safe PIN comparison** — uses `crypto.timingSafeEqual` on binary buffers
- **Process tree killing** — `/stop` uses `taskkill /T /F` (Windows) to kill all children

## Quick Start

### 1. Prerequisites

- Node.js ≥ 18
- Antigravity CLI (`agy.exe`) installed
- A Telegram bot token from [@BotFather](https://t.me/BotFather)
- Your Telegram Chat ID from [@userinfobot](https://t.me/userinfobot)

### 2. Install

```bash
git clone https://github.com/Alberte88/Dispatch-for-Antigravity.git
cd Dispatch-for-Antigravity
npm install
```

### 3. Configure

Run the interactive setup script — it handles PIN hashing for you:

```bash
node setup.js
```

This creates a `.env` file with your credentials. Your PIN is never stored in plain text.

### 4. Run

```bash
node index.js
```

The bot starts polling for messages. Open Telegram and send your PIN to unlock it.

## Architecture

```
Telegram ──► Grammy bot (long polling)
                │
                ├── SessionManager  (PIN gate, 20-min sliding timeout)
                ├── PathValidator   (blocks ../.. traversal & symlinks)
                ├── ProcessManager  (spawn agy.exe, taskkill tree on /stop)
                └── OutputBuffer    (ANSI strip, 3800-char Telegram chunks)
```

## Environment Variables

| Variable | Description |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Bot token from @BotFather |
| `ALLOWED_CHAT_ID` | Your Telegram chat ID (only you can use the bot) |
| `HASHED_PIN` | SHA-256 hex of your PIN (generated by `setup.js`) |
| `WORKSPACE_ROOT` | Root directory — workspaces must stay inside this |
| `DEFAULT_WORKSPACE` | Initial working directory for tasks |
| `ANTIGRAVITY_PATH` | Absolute path to `agy.exe` |
